kafka知识点整理

kafka-manager

1.项目地址
https://github.com/yahoo/kafka-manager
2.下载编译

git clone https://github.com/yahoo/kafka-manager.git
cd kafka-manager
sbt clean dist
执行编译的时候打包时间很长
打包完毕会生成一个zip包
解压后，修改conf/application.conf 把zkhost配置上，就可以启动了
启动脚本

  1. api 接口
    kafka又简单的api接口
    格式可以参照kafka-manager/conf/route里面的内容
    或者参考这里的https://github.com/yahoo/kafka-manager/pulls?utf8=%E2%9C%93&q=api

kafka扩容与缩容

集群leader平衡

如果扩容或者缩容，未配置auto.leader.rebalance.enable
就要手动触发leader平衡

集群内partition迁移

扩容和缩容机器后partition不会自动迁移到其他的borker上

**注意扩容或者缩容后记得调整partition的数目

集群间迁移

1.通过logstash
注意从logstash导出的数据会多2个字段

2.通过kafka-mirror-maker.sh
注意producter的配置文件加这一行 保证数据不会丢queue.enqueueTimeout.ms=-1

kafka端口链接数逐步徒增

最近一台kafka机器重启迁移了，结果发现整个kafka集群的链接数都飙升到了30万，看日志是leader发生了变化，而producter还在往旧的kafka写数据
重启agent的kafka模块后，kafka报警恢复。可是端口链接数还是没有降低多少。
后来谷歌了下，发现我用的版本是0.8.2.1,这个版本有这个bug，在0.9.0.0中修复了这个问题

0.8.2.1的kafka端口连接

如果用了0.9.0.0后,keepalive生效
